1. Employee Onboarding Checklist ↑ top

BeginnerCanvas App + FlowHR

Business Problem

New hires receive inconsistent onboarding tasks across departments, leading to missed equipment, accesses, and compliance steps.

Users & Personas

  • New Hire (read & tick)
  • HR Coordinator (configure & monitor)
  • Hiring Manager (assign tasks)

Data Model (Dataverse)

  • nh_Employee (Name, Start Date, Manager, Dept)
  • nh_OnboardingTask (Title, Owner, Due, Status) — N:1 to Employee
  • nh_Template (Role-based task templates) — 1:N TaskLine

Security Model

  • Security Roles: HR Admin, Manager, Employee
  • Row-level: Employee sees own; Manager sees direct reports via Business Unit

UX / App Design

  • Home screen: Welcome banner + % progress gauge
  • Task list with category filter (IT, Payroll, Compliance)
  • Detail screen with attachments + sign-off button

Automation Design

  • Trigger: New Employee row created → instantiate template tasks
  • Daily flow: send digest of overdue tasks to Manager
  • On all-complete: notify HR via Teams

Integrations / Connectors

  • Outlook (welcome email), Teams (chat post), SharePoint (policy docs), Azure AD (user lookup)

Key Expressions / Logic

// Canvas: progress %
RoundUp(
  CountIf(MyTasks, Status.Value="Complete") / CountRows(MyTasks) * 100, 0)

Non-functional

  • Delegation: filter Tasks by Owner = User().Email server-side
  • DLP: Business connectors only (no consumer)

ALM & Governance

  • Unmanaged in Dev, managed solutions to Test/Prod
  • Environment Variables: HR Teams URL, Template Library Site URL

Complexity

S — 3–5 days

Implementation Plan

  1. Create Dataverse solution + tables
  2. Build template seeding logic
  3. Design 3-screen Canvas app
  4. Build instantiation flow
  5. Build digest flow
  6. Add security roles
  7. UAT with HR pilot group
  8. Export managed solution → Test → Prod

Test Scenarios

  • New hire created → tasks instantiated from correct role template
  • Manager sees only direct reports
  • Overdue digest sent at 9am local time

Pitfalls & Best Practices

  • Avoid using SharePoint lists for >5000 tasks → use Dataverse
  • Never hard-code Teams URLs → environment variables
  • Pre-create role templates before go-live to avoid empty checklists

Enhancements

  • Add Copilot chat to answer "What's next?" using onboarding KB
  • Power BI exec dashboard on time-to-productivity

2. Field Facility Inspection (Offline Capable) ↑ top

IntermediateCanvas (Offline) + FlowFacilities

Business Problem

Inspectors visit sites without reliable connectivity; paper forms cause delays, lost photos, and inconsistent scoring.

Users & Personas

  • Inspector (mobile, offline)
  • Facilities Manager (review & sign off)
  • Auditor (read-only)

Data Model

  • fac_Site, fac_Inspection (Header), fac_InspectionItem (lines with score, photo, geo)
  • 1:N Site→Inspection, 1:N Inspection→Item

Security Model

  • Inspectors see their assigned sites only (BU + team-based sharing)
  • Field security on Score & Notes for auditor read-only

UX / App Design

  • Splash + offline sync indicator
  • Site list (LoadData/SaveData cache)
  • Inspection form: photo, signature, GPS, sliders
  • Pending Sync screen

Automation Design

  • On submit (online): flow validates required photos, generates PDF, emails manager
  • Critical findings → escalate to Safety Lead within 1 hour (SLA timer)
  • Retry policy: exponential, 5 attempts

Integrations

  • OneDrive (PDF archive), Outlook, Teams, Azure Blob (photo storage), Power BI (heatmap)

Key Expressions

// Cache for offline
SaveData(colInspections, "InspectionsCache");
// On startup, restore
LoadData(colInspections, "InspectionsCache", true);
// Submit when online
If(Connection.Connected, ForAll(colPending, Patch(Inspections, ...)))

Non-functional

  • Local cache <100 items / 50MB per device
  • Image compression to 1024px before upload
  • Concurrency: optimistic — last-write-wins with audit log

ALM & Governance

  • Solution-aware Canvas app; connection references for SharePoint & Outlook

Complexity

M — 2–3 weeks

Implementation Plan

  1. Design data model + indexes
  2. Build inspection schema with conditional questions
  3. Implement offline cache pattern
  4. Build sync queue + retry
  5. PDF generation flow (Word template)
  6. Critical-finding escalation flow
  7. Field pilot (3 sites)
  8. Roll out via Intune-managed devices

Test Scenarios

  • Submit inspection in airplane mode → resync when online
  • Required photo missing → block submission
  • Critical defect → Safety Lead notified <1h

Pitfalls & Best Practices

  • Avoid storing photos in Dataverse columns; use Files/Blob
  • Test SaveData on actual mobile (not Studio)
  • Add per-record GUID to prevent duplicate submission

Enhancements

  • AI Builder image classifier (e.g. detect fire-extinguisher tag)
  • QR code linking to asset record

  4. Build PDF flow

Users & Personas

  • Field Rep, Sales Ops, Order Desk

Data Model

  • sl_Account, sl_Order (header), sl_OrderLine, sl_Product (cached locally)

Security Model

  • Reps see own accounts (Owner team), Sales Ops sees all in territory (BU)

UX

  • Account list (offline)
  • Product catalog with image & price (cached)
  • Order builder + signature + total
  • Sync queue with progress

Automation

  • On sync: validate stock via SQL connector, create order in ERP, email confirmation
  • Failure → push to error log table + Teams alert to Order Desk

Integrations

  • SAP / D365 F&O, SQL Server (catalog cache), Outlook, OneDrive

Key Expressions

// Cache catalog once per day
If(DateDiff(varLastCache, Today())>0,
   ClearCollect(colProducts, Products);
   SaveData(colProducts, "ProductsCache");
   Set(varLastCache, Today()))

Non-functional

  • Delegation on Product table by category
  • Image lazy-load (thumbnail)
  • Concurrency: server timestamp check before patching

ALM

  • Connection references for SAP & SQL; environment variables for ERP endpoint

Complexity

L — 4–6 weeks

Implementation Plan

  1. Design data + sync model
  2. Build catalog caching strategy
  3. Build offline order entry screens
  4. Build sync & conflict resolution flow
  5. ERP integration (custom connector)
  6. Error logging + admin app
  7. Pilot with 5 reps
  8. Phased rollout

Test Scenarios

  • Offline order with 20 lines syncs successfully
  • Catalog price changed → cached vs live diff handled
  • ERP timeout → order queued + retried

Pitfalls

  • Don't load 100k products into a collection — paginate / filter by territory
  • Watch out for SAP rate-limit; batch calls
  • Always include a client-generated GUID for idempotency

Enhancements

  • AI Builder OCR for paper order forms
  • Predictive next-best-product suggestion

Things stop falling through the cracks. Managers only hear about problems, not every single open item.

Power Automate Flow (Scheduled + Dataverse/SharePoint + Teams + Outlook)

Complexity

  • Always store who got escalated to and when — for audit and to tune SLAs later
  • Don't send a Teams nudge during the user's local night hours — schedule by their time zone
