Text Functions
String Inspection & Extraction
LenINTLen(Text) → NumberEx: Len("Power Fx") → 8Character count of a string.⭐ Len is for strings; CountRows is for tables — frequently confused in interviews.
FindFind(FindText, WithinText [,StartNum]) → NumberEx: Find("@", Email.Text) → 1-based position of @; Blank if not foundReturns 1-based position of a substring within text. Returns Blank if not found.
StartsWithStartsWith(Text, StartText) → BooleanEx: StartsWith(Title.Text, "Mgr")True if Text begins with StartText. Case-insensitive.
EndsWithEndsWith(Text, EndText) → BooleanEx: EndsWith(FileName, ".pdf")True if Text ends with EndText. Case-insensitive.
LeftINTLeft(Text, NumChars) → TextEx: Left("Power Fx", 5) → "Power"Returns leftmost N characters.
RightRight(Text, NumChars) → TextEx: Right("Power Fx", 2) → "Fx"Returns rightmost N characters.
MidINTMid(Text, StartNum, NumChars) → TextEx: Mid("Power Fx", 7, 2) → "Fx"Returns substring from a 1-based position.⭐ Combine Mid + Find to extract parts of structured strings like "Code:Value".
SplitINTSplit(Text, Separator) → Table{Result}Ex: Split("a,b,c", ",") → 3-row table with column "Result"Splits text into a single-column table (column = Result).⭐ Use with ForAll or Gallery Items to process comma-separated values stored in a text field.
UnicodeUnicode(Text) → NumberEx: Unicode("A") → 65Returns the Unicode code point of the first character.
CharChar(Number) → TextEx: Char(10) → newline · Char(9) → tab · Char(65) → "A"Returns the character for a given code point.⭐ Char(10) is the only way to embed a line break inside a Label or multi-line TextInput.
Transformation, Encoding & Format Conversion
LowerLower(Text) → TextEx: Lower("HELLO") → "hello"Converts all characters to lowercase.
UpperUpper(Text) → TextEx: Upper("hello") → "HELLO"Converts all characters to uppercase.
ProperProper(Text) → TextEx: Proper("power fx") → "Power Fx"Capitalises the first letter of each word.
TrimINTTrim(Text) → TextEx: Trim(" Hi there ") → "Hi there"Removes leading, trailing, and extra interior spaces.⭐ Trim vs TrimEnds: Trim collapses all excess spaces; TrimEnds only strips leading/trailing spaces.
TrimEndsTrimEnds(Text) → TextEx: TrimEnds(" Hi there ") → "Hi there"Strips only leading and trailing spaces; preserves interior spacing.
ReplaceReplace(Text, StartNum, NumChars, NewText) → TextEx: Replace("Power", 1, 5, "Canvas") → "Canvas"Replaces characters at a specific position and length. Position-based.
SubstituteINTSubstitute(Text, OldText, NewText [,InstanceNum]) → TextEx: Substitute("a-b-c", "-", "/") → "a/b/c"Replaces a substring by value. InstanceNum targets a specific occurrence.⭐ Replace uses position; Substitute uses value — classic interview distinction.
PlainTextPlainText(HTMLText) → TextEx: PlainText("<b>Hi</b>") → "Hi"Strips HTML tags and decodes entities. Use to sanitise RichTextEditor output.
EncodeHTMLEncodeHTML(Text) → TextEx: EncodeHTML("<b>") → "&lt;b&gt;"Encodes HTML special characters for safe rendering in HTML context.
EncodeUrlEncodeUrl(Text) → TextEx: EncodeUrl("hello world") → "hello%20world"URL-encodes a string. Required before embedding dynamic values in Launch() URLs.
TextINTText(Value [,FormatString]) → TextEx: Text(Today(), "dd-mmm-yyyy") → "01-Jun-2026" · Text(12500.5, "$#,##0.00")Converts number, date, or datetime to a formatted string.⭐ Key format tokens: dd/mm/yyyy · hh:mm AM/PM · $#,##0.00 · 0.00%. Text() is the go-to for all display formatting.
ValueINTValue(Text [,LanguageTag]) → NumberEx: Value("1,234.56") → 1234.56Parses text to a number. Respects locale decimal/thousand separators.⭐ Always wrap TextInput.Text in Value() before arithmetic to avoid type errors.
ConcatINTConcat(Table, Formula [,Separator]) → TextEx: Concat(Tags, Name, ", ") → "Tag1, Tag2"Evaluates formula on each table row and joins results into one string.⭐ Concat (table→text) vs Concatenate (text args). Use Concat to build comma-separated display lists.
ConcatenateDEPConcatenate(Text1, Text2, …) → TextEx: prefer First.Text & " " & Last.TextJoins text strings. Prefer the & operator — shorter and more readable.
Regex – Pattern Matching

Options: Contains, Complete, BeginsWith, EndsWith, IgnoreCase, Multiline.⭐ Built-in patterns save writing raw regex: Match.Email · Match.Digits · Match.Letter · Match.MultipleLetters.
MatchMatch(Text, Pattern [,Options]) → RecordEx: Match("AB-123", "\d+").FullMatch → "123"Returns the first regex match as a record: FullMatch, SubMatches (table), StartMatch (position).

Math & Statistics
Rounding, Arithmetic & Scientific
AbsAbs(Number) → NumberEx: Abs(-5) → 5Absolute value — removes the sign.
RoundINTRound(Number, Decimals) → NumberEx: Round(3.567, 2) → 3.57 · Round(2.5, 0) → 3Rounds half-away-from-zero.⭐ Round(2.5)=3, RoundDown(2.9)=2, RoundUp(2.1)=3 — common interview comparison.
RoundUpRoundUp(Number, Decimals) → NumberEx: RoundUp(2.1, 0) → 3Always rounds away from zero regardless of the decimal portion.
RoundDownRoundDown(Number, Decimals) → NumberEx: RoundDown(2.9, 0) → 2Always rounds toward zero (truncates toward zero).
CeilingCeiling(Number [,Significance]) → NumberEx: Ceiling(4.1) → 5 · Ceiling(7, 5) → 10Rounds up to the nearest multiple of Significance (default 1).
FloorFloor(Number [,Significance]) → NumberEx: Floor(4.9) → 4 · Floor(7, 5) → 5Rounds down to the nearest multiple of Significance (default 1).
IntInt(Number) → NumberEx: Int(3.9) → 3 · Int(-3.2) → -4Rounds toward negative infinity (true floor for all real numbers).
TruncTrunc(Number [,Decimals]) → NumberEx: Trunc(-3.9) → -3 · Trunc(3.9) → 3Truncates toward zero regardless of sign.⭐ Int(-3.2) = -4 but Trunc(-3.2) = -3 — subtle negative-number difference.
ModMod(Dividend, Divisor) → NumberEx: Mod(10, 3) → 1 · Mod(8, 2) → 0 (even check)Remainder after division. Use Mod(n,2)=0 to detect even numbers or wrap sequences.
PowerPower(Base, Exponent) → NumberEx: Power(2, 10) → 1024 (same as 2^10)Raises Base to Exponent. Equivalent to the ^ operator.
SqrtSqrt(Number) → NumberEx: Sqrt(144) → 12Square root.
ExpExp(Number) → NumberEx: Exp(1) → 2.71828 (e)Returns e raised to the given power (e^x).
LnLn(Number) → NumberEx: Ln(Exp(1)) → 1Natural logarithm (base e). Inverse of Exp.
LogLog(Number [,Base]) → NumberEx: Log(1000, 10) → 3 · Log(8, 2) → 3Logarithm. Default base is 10 when omitted.
PiPi() → NumberEx: Pi() → 3.14159265358979…Returns the mathematical constant π.
RandRand() → NumberEx: Rand() → 0.0 to 0.9999… (recalculates on every evaluation)Random decimal in [0,1). Recalculates on every formula evaluation — not cached.
RandBetweenRandBetween(Low, High) → NumberEx: RandBetween(1, 100) → random integer 1–100 inclusiveRandom integer between Low and High inclusive.

AverageAverage(Table, Formula) or Average(N1, N2, …) → NumberEx: Average(Scores, Value)Mean of values. Table form is delegable on Dataverse.

CountACountA(Table, Formula) → NumberEx: CountA(Tasks, Title)Counts rows where the formula result is non-blank (any type).
CountIfINTDCountIf(Table, Condition) → NumberEx: CountIf(Tasks, Status = "Done")Counts rows matching a condition. Delegable on Dataverse.⭐ Use CountIf instead of CountRows(Filter(…)) — CountIf is delegable on Dataverse for large datasets.
CountRowsINTNDCountRows(Table) → NumberEx: CountRows(colCart) → number of items in collectionTotal number of rows in a table or collection. Never delegable.⭐ For accurate row counts on large Dataverse tables, use CountIf(Table, true) — it is delegable.

Logical, Conditional & Error Handling
Conditionals, Boolean Logic & Blank / Type Checks
IfINTIf(Condition, ThenResult [,ElseResult]) → AnyEx: If(Score >= 60, "Pass", "Fail")Returns ThenResult when true, ElseResult when false. Can be chained: If(c1,r1, c2,r2, default).⭐ Switch is cleaner for 3+ branches. If re-evaluates each condition; Switch evaluates the formula once.
SwitchINTSwitch(Formula, Match1, Result1 [,…] [,Default]) → AnyEx: Switch(Status, "New","🔴", "Done","🟢", "⚫")Matches formula against a list of values and returns the corresponding result. More readable than nested If.⭐ Switch evaluates the formula once; nested If re-evaluates each branch condition — Switch is cheaper for 3+ cases.
And / &&And(Cond1, Cond2, …) → BooleanEx: And(Age > 18, HasID) or Age > 18 && HasIDTrue only when ALL conditions are true. Short-circuits on the first false.
Or / ||Or(Cond1, Cond2, …) → BooleanEx: Or(Role = "Admin", Role = "Owner")True when ANY condition is true. Short-circuits on the first true.
Not / !Not(Condition) → BooleanEx: Not(IsBlank(SearchBox.Text))Inverts a boolean value.
IsBlankINTIsBlank(Value) → BooleanEx: IsBlank(TextInput1.Text)True if value is blank or an empty string ("").⭐ IsBlank checks single values; IsEmpty checks tables — both appear in every canvas app validation pattern.
IsEmptyINTIsEmpty(Table) → BooleanEx: IsEmpty(colCart) · IsEmpty(Filter(Tasks, Status="Open"))True if the table or collection has zero rows.⭐ IsEmpty(Filter(…)) is more efficient than CountRows(Filter(…)) = 0 for delegation awareness.
BlankBlank() → BlankEx: Set(varUser, Blank())Returns an explicit blank value. Use to reset variables or clear optional lookup fields.
CoalesceINTCoalesce(Value1, Value2, …) → AnyEx: Coalesce(UserInput.Text, "Default value")Returns the first non-blank value in the list.⭐ Cleaner than If(IsBlank(x), fallback, x). Essential for providing safe defaults on optional fields.
IsNumericIsNumeric(Text) → BooleanEx: IsNumeric("123abc") → false · IsNumeric("42") → trueTrue if text can be parsed as a valid number. Use for input validation before calling Value().
IsTypeIsType(Record, TypeName) → BooleanEx: If(IsType(item.Owner, Contacts), AsType(item.Owner, Contacts).FullName)Checks whether a polymorphic record is of a specific Dataverse entity type. Always use before AsType.
BooleanBoolean(TextOrNumber) → BooleanEx: Boolean("yes") → true · Boolean(0) → falseParses "true"/"false"/"yes"/"no"/1/0 strings to Boolean. Useful for SharePoint Yes/No fields stored as text.
Error Handling
IfErrorINTIfError(Value, Replacement [,Value2, Replacement2, …]) → AnyEx: IfError(1 / Qty, 0) · IfError(Patch(…), Notify("Error: " & FirstError.Message))Returns Value; if it is an error, returns Replacement instead.⭐ Essential for safe division and Patch operations. Always wrap Patch in IfError in production apps.
IsErrorIsError(Value) → BooleanEx: If(IsError(Value(TextInput.Text)), "Not a number", "")True if the expression produced an error. Use to show contextual validation messages.
ErrorError({Kind: ErrorKind.xxx, Message: "…"}) → ErrorEx: Error({Kind: ErrorKind.Custom, Message: "Age is required"})Raises a custom error that propagates up and is caught by an enclosing IfError.

Columns: Record, Column, Message, Error (ErrorKind).⭐ Use in OnFailure of SubmitForm to surface field-level validation errors from Dataverse business rules.
ErrorKind values: Conflict · ConstraintViolation · CreatePermission · DeletePermission · EditPermission · GeneratedValue · None · NotFound · ReadPermission · Sync · Unknown · Validation · Custom

TodayINTToday() → DateEx: Today() → current date at midnightReturns today's date with no time component. Use for date comparisons and arithmetic.
DateDate(Year, Month, Day) → DateEx: Date(2026, 12, 31) · Date(2026, 13, 1) → 1 Jan 2027 (overflow wraps)Constructs a Date from year/month/day integers. Month and Day overflow wraps correctly.
TimeTime(Hour, Minute, Second) → TimeEx: Time(9, 30, 0) → 09:30:00 · Add to a Date with + to get DateTimeConstructs a Time value. Add to a Date using the + operator to build a full DateTime.
DateTimeValueDateTimeValue(Text [,LanguageTag]) → DateTimeEx: DateTimeValue("2026-06-01T09:00:00")Parses a text string into a DateTime. Supports ISO 8601 and locale-specific formats.
DateValueDateValue(Text [,LanguageTag]) → DateEx: DateValue("01/06/2026", "en-GB")Parses text to a Date. LanguageTag resolves ambiguous formats (mm/dd vs dd/mm).
TimeValueTimeValue(Text [,LanguageTag]) → TimeEx: TimeValue("9:30 AM")Parses a text string to a Time value.

Table Operations

SearchNDINTSearch(Table, SearchText, Col1 [,Col2, …]) → TableEx: Search(Products, SearchBox.Text, "Name", "SKU")Case-insensitive substring search across named text columns. Always client-side.⭐ Search is NEVER delegable. Use Filter + StartsWith for delegable text search on large Dataverse tables.

SortINTSort(Table, Formula [,Order]) → TableEx: Sort(Products, Price, SortOrder.Descending)Sorts by a formula. SortOrder.Ascending (default) or SortOrder.Descending.⭐ Sort on a computed formula = NOT delegable. SortByColumns on a real column = delegable.

ForAll, Sequence, Grouping & Table Construction
ForAllINTNDForAll(Table, Formula) → TableEx: ForAll(colItems, Patch(Orders, Defaults(Orders), {Name: Name, Qty: Qty}))Evaluates a formula for every row; returns the resulting table. Always client-side.⭐ ForAll is non-delegable. For delegable batch writes use Patch(DataSource, Table) instead.
SequenceINTSequence(Count [,Start] [,Step]) → Table{Value}Ex: Sequence(5) → [1,2,3,4,5] · Sequence(5, 0, 10) → [0,10,20,30,40]Generates a single-column table of sequential numbers.⭐ Sequence is the cleanest way to build custom paging controls or repeat operations N times.

RenameColumnsRenameColumns(Table, OldName, NewName [,…]) → TableEx: RenameColumns(Result, "cr123_name", "ProductName", "cr123_price", "Price")Renames columns. Essential for mapping Dataverse internal/publisher-prefixed names to readable display names.

Collections & Patch (Data Mutation)
Collection Operations – Collect · ClearCollect · Clear · Remove · RemoveIf · Update · UpdateIf
CollectINTCollect(Target, Item1 [,Item2, …]) → TableEx: Collect(colCart, {Name: Product.Name, Qty: 1, Price: Product.Price})Appends records to a collection. Does NOT clear existing data first.⭐ Collect ADDS; ClearCollect REPLACES — the most common interview mix-up on collection functions.
ClearCollectINTClearCollect(Target, Table) → TableEx: ClearCollect(colProducts, Filter(Products, Active = true))Clears the collection first then loads new data. Standard pattern for caching on app start.⭐ Put ClearCollect in App OnStart (inside Concurrent) to pre-load data and reduce screen-load latency.
ClearINTClear(Collection) → VoidEx: Clear(colCart)Empties all rows from the collection. The collection variable still exists.⭐ Use Clear to reset a shopping cart or a multi-step selection buffer between sessions.
RemoveRemove(Source, Record1 [,Record2, …] [,All]) → VoidEx: Remove(colCart, ThisItem)Removes specific records from a collection or data source by record reference. All removes all matching.
RemoveIfRemoveIf(Source, Condition [,Condition2, …]) → VoidEx: RemoveIf(colCart, Qty = 0)Removes all rows matching the condition. Works on both collections and data sources.
UpdateUpdate(Collection, OldRecord, NewRecord [,All]) → VoidEx: prefer UpdateIf or Patch insteadReplaces an entire record in a collection. Rarely used in modern apps — prefer UpdateIf or Patch.
UpdateIfUpdateIf(Collection, Condition, ChangeRecord [,…]) → VoidEx: UpdateIf(colCart, ID = selectedID, {Qty: NewQty})Updates matching rows in a local collection. Does not work on server data sources — use Patch for those.
Patch, Defaults, Refresh & Errors – Server Data Source Write Operations
PatchINTPatch(DataSource, BaseRecord, ChangeRecord [,…]) → RecordEx (create): Patch(Accounts, Defaults(Accounts), {Name: "Acme", Status: "Active"})
Ex (update): Patch(Accounts, LookUp(Accounts, ID = varID), {Status: "Closed"})		Creates or updates a record by merging ChangeRecord into BaseRecord.⭐ Top interview topic. Defaults(DataSource) = create new; existing record = update. Always wrap in IfError.
Patch (bulk)Patch(DataSource, Table) → TableEx: Patch(Orders, colPendingOrders)Upserts all rows of a table into the data source in a single operation. Far more efficient than ForAll + Patch per row.
DefaultsINTDefaults(DataSource) → RecordEx: Patch(Tasks, Defaults(Tasks), {Title: txtTitle.Text, AssignedTo: User().Email})Returns a blank record with default values for the data source. Use as BaseRecord to tell Patch to create a new record.⭐ Defaults is the only way to signal "create new" to Patch — it provides the empty base row.
RefreshRefresh(DataSource) → VoidEx: Refresh(Accounts)Forces the data source to reload from the server, clearing the local cache.
ErrorsINTErrors(DataSource [,Record]) → TableEx: Errors(Accounts, EditForm1.LastSubmit)Returns error details from the last data operation. Columns: Record, Column, Message, Error (ErrorKind).⭐ Call in OnFailure of SubmitForm to surface column-level validation errors from Dataverse business rules.
Form Functions
SubmitForm · ResetForm · NewForm · EditForm · ViewForm
SubmitFormINTSubmitForm(FormControl) → VoidEx: SubmitForm(EditForm1)Validates all data cards then calls Patch on the data source. OnSuccess / OnFailure fire after.⭐ SubmitForm is declarative — use OnSuccess to navigate away and Errors() in OnFailure to display messages.
ResetFormINTResetForm(FormControl) → VoidEx: ResetForm(EditForm1)Resets all data card fields to their original values and clears unsaved changes.⭐ ResetForm does NOT clear the Item property. Combine with Set(varItem, Blank()) to fully reset a form.
NewFormNewForm(FormControl) → VoidEx: NewForm(EditForm1)Sets the form to New mode — all fields blank, ready to create a new record.

User-Defined Types, Functions & Named Formulas GA
App.Formulas — Named Formulas
Named formulaName = Expression;Ex: BGColor = ColorValue(Param("BackgroundColor"));Defines an immutable, always-available, auto-recalculating value inside the App.Formulas property. Replaces the OnStart-variable pattern — no timing dependency.⭐ The Excel-named-cell equivalent. Calculation is deferred until first use — a formula only needed on screen2 isn't computed until screen2 is shown.
Chained formulasA = expr1; B = A + 1;Ex: UserInfo = LookUp(Users, 'Primary Email' = User().Email); UserTitle = UserInfo.Title;Named formulas can reference each other in any order (no circular references) and are evaluated in parallel — order in App.Formulas doesn't matter.
Limitations: can't call behavior functions (Set, Notify, Patch, Collect…) or otherwise cause side effects, and can't form a circular reference (a = b; b = a; is invalid).
User-Defined Functions (UDF)
Pure UDFINTFnName(Param: Type, …): ReturnType = Formula;Ex: LibraryGenre(SelectedGenre: Text): LibraryType = Filter(Library, Genre = SelectedGenre);A named formula with parameters — call it like a built-in function, e.g. LibraryGenre("Fiction").⭐ Every parameter and the return value must be typed: a built-in type name, a data source name, or a type built with Type().
Behavior UDFFnName(Param: Type): Void = { Formula1; Formula2 };Ex: Spend(Amount: Number): Void = { Set(Savings, Savings - Amount); Set(Spent, Spent + Amount) };Wrap the body in { } to allow Set/Notify/Patch-style side effects. Return type is Void when nothing is returned.
⚠️ Recursion isn't yet supported. A record-type parameter's fields must be a proper subset of the type definition — extra fields error out (e.g. passing {Title:"x", Published:2001} where the type only declares Title fails).
User-Defined Types (UDT) — Type() & RecordOf()
TypeINTTypeName := Type(TypeSpecification)Ex: BookType := Type({ Title: Text, Author: Text, Published: Number });Declares a named, reusable type for UDF signatures, ParseJSON, and untyped-object conversions. Use :=, not =.⭐ A record type spec mirrors record-literal syntax with type names instead of values: {Name:"Jane"} → Type({Name: Text}).
Table typeTableType := Type( [ RecordType ] )Ex: LibraryType := Type( [ BookType ] );Wrap a record type in square brackets for the type of a table of those records — square brackets here, not the Table() function.
RecordOfRecordOf(TableType) → TypeEx: BookType := Type( RecordOf( LibraryType ) );Extracts the single-record type from an existing table type. Valid only inside Type() — use when you have the table type first and need the row type.
ParseJSON + UDTParseJSON(JsonText, UDT) → RecordEx: ParseJSON(jsonText, BookType) → typed record, no per-field IsType/AsType neededPassing a user-defined type as ParseJSON's second argument returns a fully typed record directly.
✅ Generally available (graduated from experimental preview) — usable in canvas apps' App.Formulas, custom connectors, and component libraries. Declare named types like Customer, OrderLine, InvoiceHeader once and reuse them everywhere instead of repeating inline type specs.
Delegation Guide
Which Functions Delegate? – Dataverse vs SharePoint
FunctionDataverseSharePoint    Notes
Filter✔ Delegable✔ Delegable  · Simple predicates (=, <, >, And, Or, in). Complex/computed expressions fall back to client.
LookUp✔ Delegable✔ Delegable  · Single-record fetch — fully delegable with simple conditions.
SortByColumns✔ Delegable✔ Delegable  · Sort on real column names. Computed expressions are non-delegable.
Sort (formula)✘ Not Delegable✘ Not Delegable  · Sort on a computed formula is always client-side.
Search✘ Not Delegable✘ Not Delegable  · Always client-side. Use Filter + StartsWith for delegable text search on Dataverse.
StartsWith (in Filter)✔ Delegable✘ Not Delegable  · Dataverse: delegable text prefix. SharePoint: client-side only.
CountIf✔ Delegable✘ Not Delegable  · Dataverse: delegable. SharePoint: capped at 500 rows.
Sum / Average / Max / Min✔ Delegable✘ Not Delegable  · Table aggregate form delegates on Dataverse only.
CountRows✘ Not Delegable✘ Not Delegable  · Never delegable. Use CountIf(Table, true) on Dataverse for accurate large counts.
Distinct~ Partial✘ Not Delegable  · Dataverse: delegable on simple columns only. Computed formula = non-delegable.
In (membership)✔ Delegable~ Partial  · Filter(Table, ID in colIDs) — delegable on Dataverse for choice/lookup columns.
IsBlank (in Filter)✔ Delegable~ Partial  · Dataverse: delegable. SharePoint: may be non-delegable depending on column type.
ForAll✘ Not Delegable✘ Not Delegable  · Always client-side. Use Patch(DataSource, Table) for delegable bulk writes instead.
⚠️ Delegation limit default: 500 rows (raise to 2 000 in Settings → Data row limit). Data beyond the limit is silently dropped. Use Filter + SortByColumns as the safest delegable combination for large tables.
Common Pattern Recipes
Data & Collection Patterns
1. Create a new Dataverse record (Patch + Defaults)
IfError( Patch(Accounts, Defaults(Accounts), {Name: txtName.Text, Status: "Active"}), Notify("Error: " & FirstError.Message, NotificationType.Error) ); Notify("Account created!", NotificationType.Success);
Wrap every Patch in IfError — Dataverse business rule violations surface as errors, not exceptions.
2. Delegable text search with StartsWith
// Gallery Items (delegates on Dataverse): Filter(Contacts, StartsWith(FullName, SearchBox.Text) || StartsWith(Email, SearchBox.Text))
StartsWith delegates on Dataverse. Search() does NOT — avoid Search() on tables with 500+ rows.
3. Cache data on app start with Concurrent
// App OnStart: Concurrent( ClearCollect(colProducts, Filter(Products, Active = true)), ClearCollect(colCategories, Distinct(Products, Category)), Set(gblCurrentUser, LookUp(Employees, Email = User().Email)) );
Concurrent() runs multiple ClearCollect calls in parallel — significantly faster than sequential.
4. Cascading dropdown
// ddCategory Items: Distinct(Products, Category) // ddProduct Items: Filter(Products, Category = ddCategory.Selected.Result)
Filter the child dropdown on the parent's Selected value. Delegable on Dataverse.
5. Shopping cart upsert (add or increment)
If(CountIf(colCart, ID = ThisItem.ID) = 0, Collect(colCart, {ID: ThisItem.ID, Name: ThisItem.Name, Qty: 1, Price: ThisItem.Price}), UpdateIf(colCart, ID = ThisItem.ID, {Qty: Qty + 1}) ); // Cart total: Sum(colCart, Qty * Price)
CountIf checks existence; Collect adds; UpdateIf increments — clean upsert on a collection.
6. Batch write a collection to Dataverse
IfError( Patch(Orders, colPendingOrders), Notify("Batch failed: " & FirstError.Message, NotificationType.Error) ); Clear(colPendingOrders);
Single Patch(DataSource, Table) is far more efficient than ForAll + Patch per row.
UX & Formula Patterns
7. Pass data between screens via Navigate context
// Button OnSelect (Screen A): Navigate(DetailScreen, ScreenTransition.Fade, {locRecord: ThisItem, locMode: "Edit"}) // DetailScreen reads locRecord.Name, locRecord.ID etc. directly
ContextRecord in Navigate sets UpdateContext variables on the target screen — the cleanest inter-screen data transfer.
8. Loading spinner during data fetch
// Button OnSelect: UpdateContext({locLoading: true}); Concurrent( ClearCollect(colData, MyDataSource), Set(gblConfig, LookUp(Config, Key = "App")) ); UpdateContext({locLoading: false}); // Spinner Visible: locLoading
Toggle locLoading to show/hide a spinner while data loads in the background.
9. With() to avoid repeated expensive LookUp
// Instead of two LookUp calls: With({emp: LookUp(Employees, ID = Gallery1.Selected.ID)}, emp.Name & " – " & emp.Department & " – " & emp.Email)
With() evaluates the LookUp once and aliases the result — cheaper and more readable than repeating.
10. Dynamic label color by status (Switch + RGBA)
// Label Color property: Switch(ThisItem.Status, "Active", RGBA(0, 128, 0, 1), "Pending", RGBA(255, 165, 0, 1), "Closed", RGBA(200, 0, 0, 1), RGBA(128, 128, 128, 1))
Switch + RGBA for status-based colour coding — no separate icon or image column needed.
11. Age calculation and deadline countdown
// Age in full years: DateDiff(BirthDatePicker.SelectedDate, Today(), TimeUnit.Years) // Days until deadline (negative = overdue): DateDiff(Today(), DeadlineDatePicker.SelectedDate, TimeUnit.Days)
Always specify TimeUnit explicitly for clarity and to avoid ambiguity in code reviews.
12. Build a comma-separated tag list from multi-select
// Display selected tags from a checkbox gallery: Concat(Filter(colTags, Selected), Name, ", ") // Deep-link pattern: pass ID via Launch, read via Param // Launch(appURL, "id", ThisItem.ID) → Set(gblID, Param("id")) in App OnStart
Concat(Table, Formula, Separator) is the cleanest way to join multi-selected items into a display string.
