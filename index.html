Projects

Selected engagements — context, architecture, decisions, and outcomes

01

ABB Cloud Transformation — Power Platform & ALM

Capgemini · 2025 – Present
Context
ABB operated a legacy SharePoint 2010 environment with 50+ InfoPath forms, Nintex workflows, and no structured ALM governance. Client required modernisation to Power Platform with a sustainable, scalable ALM foundation.
Architecture
Multi-environment ALM structure (Dev/Test/UAT/Prod) with managed solution packaging, CI/CD via Azure DevOps, Copilot Studio intelligent agent layer, and Model-Driven Power Apps replacing InfoPath forms on Dataverse.
Tools
Microsoft Copilot Studio AI Builder Power Platform ALM Azure DevOps Power Platform Build Tools Power Automate Model-Driven Power Apps Dataverse SharePoint Online
Decisions
Chose managed solutions over unmanaged to enforce environment promotion integrity and prevent customisation drift. Selected Model-Driven Apps over Canvas for form-heavy use cases requiring Dataverse security model alignment. Used connection references and environment variables to enable environment-agnostic pipelines.
Outcome
50+ InfoPath forms modernised; Nintex workflows migrated to Power Automate (40% efficiency gain); CI/CD pipelines operational; Copilot Studio agents deployed for enterprise AI automation workflows.
02

H1 Enterprise Cloud Migration

Capgemini · Nov 2020 – Mar 2025
Context
H1 project required migrating 500+ GB of business-critical data from legacy systems to Microsoft 365, with parallel redesign of manual form processes causing productivity bottlenecks and compliance risk.
Architecture
Phased M365 migration with PowerShell-automated content migration, metadata preservation, and permissions remapping; Canvas App replacing legacy forms with Azure SQL as primary data store; Power Automate handling multi-level approval orchestration with GDPR-compliant data scoping.
Tools
PowerShell Power Apps Canvas Azure SQL Power Automate Microsoft 365 SharePoint Online
Decisions
Chose Canvas Apps over Model-Driven as data resided in Azure SQL (not Dataverse) and required offline capability. Designed PowerShell migration scripts with delta processing to allow incremental migration without system downtime and to simplify delta reconciliation.
Outcome
Migration time reduced by 60% through automation; redesigned form-based processes improved productivity by 45%; GDPR-compliant approval workflows implemented with full audit trail.
